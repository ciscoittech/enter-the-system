# Per-Chapter Review Checklist

## Pre-Review

- [ ] Read the entire draft without making notes (get the flow)
- [ ] Note your gut reaction: did it feel like a person talking, or a textbook?

## Structure Check

- [ ] Hook opens with a specific scenario (not a general statement)
- [ ] System section shows the full workflow before details
- [ ] Walk-through steps each start with a verb
- [ ] Walk-through includes what Claude does at each step
- [ ] 2-4 prompt templates present with line-by-line explanation
- [ ] "You Got It Right When" has 3-5 concrete checks
- [ ] Prompt Pack collects all prompts with ratings

## Voice Check

- [ ] Read the first paragraph aloud — does it sound human?
- [ ] No phrases from the AI Tells list (check voice.md)
- [ ] Paragraph lengths vary (not all the same)
- [ ] At least one single-sentence paragraph for emphasis
- [ ] No "Let's dive in" / "Key takeaways" / "In conclusion"
- [ ] No hedging ("it might" / "could potentially" / "it's worth noting")
- [ ] Specific numbers where possible ("40 minutes" not "significant time")

## Jargon Check

- [ ] Search for every term in anti-jargon.md
- [ ] Each technical term has inline translation on first use
- [ ] No acronyms without expansion
- [ ] "Claude" used instead of "the model" / "the AI" / "the LLM"

## Prompt Check

- [ ] Each prompt follows the 4-part formula (Context, Input, Output, Exceptions)
- [ ] Prompts are copy-paste ready (no [placeholder] without explanation)
- [ ] Expected output described after each prompt
- [ ] Failure modes noted for each prompt
- [ ] Difficulty and effectiveness ratings assigned

## Progression Check

- [ ] No "We'll cover this in Chapter X" (forward references)
- [ ] Prior concepts referenced with brief reminders
- [ ] Complexity level matches chapter number
- [ ] New concepts introduced one at a time

## Final Check

- [ ] Word count within range (1,800-3,000)
- [ ] Would you give this to a non-technical friend? What would they struggle with?
