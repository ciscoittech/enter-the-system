# Skill Progression Map — Systems Thinking Edition

## Component Introduction Schedule

The reader builds their toolkit one component at a time. Never introduce two components in one chapter.

| Chapter | New Component | New Concept | Reader Can Now... |
|---------|--------------|-------------|-------------------|
| 00 | (none) | Systems vs tasks | Understand why prompts alone aren't enough |
| 01 | (none) | Input → Process → Output → Feedback | See any work task as a system with 4 elements |
| 02 | **Prompt** | 4-part formula, Chat vs Cowork | Write a structured prompt and get a result |
| 03 | (none) | Open vs closed loop, feedback types | Explain why a prompt without feedback is incomplete |
| 04 | **State** | Operational memory | Build a system that remembers across sessions |
| 05 | (state practice) | State patterns, decay | Build a tracker that improves over time |
| 06 | (none) | Knowledge loading, skill concept | Identify recurring context they keep re-explaining |
| 07 | **Skill** | Reusable knowledge modules | Give Claude expertise that persists across tasks |
| 08 | **Hook** | Automated guard rails | Build a system that catches mistakes before shipping |
| 09 | (integration) | Component interaction, simplicity | Combine 4 components into a real working system |
| 10 | **Connection** | APIs, MCP, CLI, web search | Pull live external data into their system |
| 11 | **Pipeline** | Multi-stage flow, quality gates | Build a multi-step workflow with checkpoints |
| 12 | (design patterns) | Loop, Gate, Router patterns | Design new systems from scratch without the book |
| 13 | (full integration) | System composition, shared state | Wire multiple systems into one operating system |
| 14 | (future-proofing) | System evolution, constraint migration | Maintain and evolve their system independently |
